내가 까먹을까봐 써두는 docker 쌩기초

2019-09-15
  • docker

# 비유하자면..

설계도 -> 만들어진거
OOP class -> instance
풀빵가게 붕어빵틀 -> 붕어빵
docker image -> container

이다.

# mariadb를 한다면...

일단 실행!

$ docker run --name maria-db \
-p 3306:3306 \
-d mariadb \
-v /Users/[username]/dev/docker/mariadb/data:/var/lib/mysql \
-v /Users/[username]/dev/docker/mariadb/conf.d:/etc/mysql/conf.d \
-e MYSQL_ROOT_PASSWORD=[당신이 원하는 root의 password] \
-character-set-server=utf8mb4 \
–collation-server=utf8mb4_unicode_ci

위의 명령을 설명하면 다음과 같다.

  • container의 이름은 maria-db 라고 한다.
  • container의 3306 포트를 host의 3306 포트로 연결한다. ( -p [local port]:[container port] )
  • container를 만들기 위한 image의 이름은 mariadb이다.
    • mariadb라는 이름의 image가 없다면 image를 docker hub(maven으로 친다면 뭐 maven central 정도 된다.)를 통해 다운로드받는다. 이는 docker pull mariadb 명령으로 직접 할 수도 있다.
    • docker pull을 사용한다면 docker hub에 없는, 네트워크에 위치한 파일이나 github를 통해 바로 다운받을 수도 있다. 그렇게 해서 다운받은 내용은 local에 저장되게 되며, 저장되어 있는한 언제든지 해당 이름을 통해 container로 만들어낼 수 있다.
    • image 이름을 정해주며 가져오라고 시킬때 특정 버전을 지정하지 않았으므로 실행 시점에서의 latest로 지정된 버전을 다운로드받게된다.
  • volume지정은 host와 container의 fs를 연결하는 선언이다. ( -v [local path]:[container path] )
    • container의 /var/lib/mysql 는 host의 /Users/[username]/dev/docker/mariadb/data 로 연결된다.
    • container의 /etc/mysql/conf.d는 host의 /Users/[username]/dev/docker/mariadb/conf.d로 연결된다.
  • 그 뒤의 구문은 환경 설정 같은 걸 명시하는 내용이다. 위에선 root의 패스워드를 정하고 character set과 collation을 지정했다.

그리고... 대충 뭐.. 기본적인 명령들은 이렇다.

# 현재 active한 모든 container의 목록을 표시한다.
$ docker ps

# 현재 생성된 모든 container의 목록을 표시한다.
$ docker ps -a

# 연결된 registry에서 image를 다운로드한다.
$ docker pull [image name]:[버전, 생략시 최신 버전]

# 지정한 컨테이너를 시작한다.
$ docker start [container name | id]

# 지정한 컨테이너를 재시작한다.
$ docker restart [container name | id]

# 컨테이너 안의 명령을 실행한다.
$ docker exec -it [container name | id] [명령]

# bash 쉘 열기
$ docker exec -it [container name | id] bash

# 컨테이너의 로그를 출력한다.
$ docker logs [container name | id]

# 마지막 500줄을 출력하면서 바로 끝나지 말고 계속 해서 log를 출력한다. tail 명령 비스므리하다.
$ docker logs --tail 500 -f [container name | id]

# 지정한 컨테이너를 중지한다.
$ docker stop [container name | id]

# 지정한 컨테이너를 삭제한다.
$ docker rm [container name | id]

# 지정한 이미지를 삭제한다.
$ docker rmi [image name | image hash]

# 저건 1개씩 하는거고...

여러 개의 docker container를 한 세트로 띄울려면 docker-compose를 사용하면 된다.

version: '3.1'

services:
  db:
    container_name: maria-db
    image: mariadb:latest
    restart: always
    ports:
      - 3306:3306
    volumes:
      - /Users/[username]/dev/docker/mariadb/data:/var/lib/mysql
      - /Users/[username]/dev/docker/mariadb/config:/etc/mysql/conf.d
    environment:
      MYSQL_ROOT_PASSWORD: [당신이 원하는 root의 password]
      TZ: Asia/Seoul
    command: -character-set-server=utf8mb4 -collation-server=utf8mb4_unicode_ci

위의 내용을 docker-compose.yml 파일로 저장하면 된다. services 밑에 다른거 적어주면 추가가 되는거니 적당히 적어주면 된다.

그리고 나서 docker-compose.yml 파일이 있는 곳에서 명령을 사용하면 된다.

# docker-compose.yml에 선언된 대로 docker container를 생성한다.
$ docker-compose up [-f `docker-compose.yml 경로`] [-p 서비스명] -d [--force-recreate]

# 시작
$ docker-compose start

# 중지
$ docker-compose stop

# docker-compose.yml에 선언된 대로 생성된 docker container를 삭제한다.
$ docker-compose down [--remove-orphans]

# 그래서 뭐가 좋은데?

옛날옛날에 이런 일이 있었다. 개발하다가 redis가 필요해졌다. 그런데 난 window 환경이네? redis를 쓰고 싶은데 난 window라 compile할 수 있는게 없어서 virtualbox를 설치해서 리눅스 깔고 하든지 해야되던 시절이 있었다. 지금이야 window 버전, 그러니까 visual studio에서 컴파일 가능한 버전이 git에서도 배포 중이다.

docker가 있다면? 고민이 필요없이 그냥 container 만들어서 띄우면 된다.

뭐가 필요해? why not? docker image만 만들 수 있다면, 그리고 존재한다면 바로 ok! 리눅스 설치고 뭐고 걱정할 필요가 없다.

이것만으로도 쓸모는 이미 증명되었다고 봐도 된다.
